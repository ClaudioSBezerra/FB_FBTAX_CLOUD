**Estado da página:**
- `clientes: Cliente[]` — lista carregada do backend
- `editando: Cliente | null` — cliente sendo editado/criado; null = Dialog fechado
- `busca: string` — termo de busca com debounce de 300ms
- `statusFiltro: 'todos' | 'ativo' | 'inativo'` — filtro de status
- `loading: boolean` — carregamento inicial
- `submitting: boolean` — submit em andamento
- `error: string | null`

**Carregamento com debounce:** Usar `useEffect` com dependências `[busca, statusFiltro]`. Dentro do efeito, criar `setTimeout` de 300ms antes de chamar o fetch. `clearTimeout` no cleanup do useEffect para cancelar chamadas pendentes. Construir URL: `/api/financeiro/clientes` + URLSearchParams com `q` e `status` conforme filtros.

**Formulário no Dialog:** Campos obrigatórios: razao_social, cnpj. Campos opcionais: email, telefone, responsavel. Campo ativo (toggle): `<input type="checkbox">` com label "Cliente ativo" — aparece apenas no modo edição (quando `editando?.id` existe). Botão "Salvar" desabilitado durante submitting.

**handleSave:** Se `editando.id` → PUT; else → POST. Após sucesso, fechar Dialog (setEditando(null)) e recarregar lista (chamar fetchClientes diretamente ou atualizar estado local com setClientes).

**Botão "Novo Cliente":** Abre Dialog com `setEditando({ razao_social: '', cnpj: '', ativo: true })` (sem id).

**Tabela:** Colunas: Razão Social, CNPJ (formatar com máscara XX.XXX.XXX/XXXX-XX), Status (Badge variant="default" para ativo, "secondary" para inativo), Ações (botão "Editar" que chama setEditando(cliente)).

**Estado vazio:** Se `clientes.length === 0` e não loading, exibir mensagem "Nenhum cliente encontrado." dentro da tabela (TableRow com TableCell colSpan={4}).

**Layout:** `div className="min-h-screen bg-gray-50 p-8"` → `div className="max-w-5xl mx-auto space-y-6"` → `Card` com header "Clientes" + botão "Novo Cliente" alinhado à direita no CardHeader → linha de busca (Input + select de status) → Table.

interface Plano {

  nome: string
  preco: number | null

**Estado da página:**
- `produtos: Produto[]` — carregados via GET /api/financeiro/produtos no mount
- `loading: boolean`
- `salvando: Record<string, boolean>` — chave é plano.id; true enquanto PUT está em andamento para esse plano específico

- `error: string | null`

**Carregamento:** useEffect sem dependências → fetch('/api/financeiro/produtos') → setProdutos(data). Ao receber produtos, popular `precoEditado` mapeando cada plano: `plano.preco != null ? String(plano.preco) : ''`.

**handleSalvarPreco(plano: Plano):** Pegar valor de `precoEditado[plano.id]`. Se string vazia → preco = null. Se não → parseFloat — se NaN → toast.error("Preço inválido") → return. Chamar PUT /api/financeiro/planos com `{id: plano.id, preco}`. Setar `salvando[plano.id] = true` antes, remover após. Após sucesso → toast.success("Preço atualizado").

**Formatação:** Input type="text" (não "number") para permitir valor vazio sem zero. Campo com `className="w-32"` para manter compacto. Exibir "R$" como prefixo antes do input usando flexbox.

**Layout principal:** `div className="min-h-screen bg-gray-50 p-8"` → `div className="max-w-4xl mx-auto space-y-6"` → heading "Produtos e Planos" → cards dos produtos.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Spoofing | ClientesPage fetch | mitigate | AuthContext monkey-patch injeta JWT em todos os fetch automaticamente; sem bypass possível no frontend |
| T-02-07 | Tampering | Campo preco em ProdutosPage | mitigate | parseFloat + verificação NaN antes de enviar PUT; string vazia resulta em null (intencional para Sob Demanda) |
| T-02-08 | Information Disclosure | Lista de clientes exposta | accept | Rota /admin/financeiro/* protegida por ProtectedRoute no frontend + withAuth no backend — dupla proteção |
| T-02-09 | Denial of Service | Debounce ausente em busca | mitigate | Debounce de 300ms via setTimeout/clearTimeout no useEffect previne disparo excessivo de requests |
| T-02-SC | Tampering | nenhum novo pacote npm neste plano | accept | Sem novas dependências — apenas componentes shadcn/ui já instalados |
</threat_model>
