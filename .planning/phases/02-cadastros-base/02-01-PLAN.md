```go
package handlers

import (
    "database/sql"
    "encoding/json"
    "net/http"
)

// Struct de request com json tags snake_case e omitempty nos opcionais
type EmpresaRequest struct {
    ID          string `json:"id,omitempty"`
    RazaoSocial string `json:"razao_social"`
    CNPJ        string `json:"cnpj"`
}

// Factory pattern — retorna http.HandlerFunc que encapsula db
func EmpresaHandler(db *sql.DB) http.HandlerFunc {
    return func(w http.ResponseWriter, r *http.Request) {
        w.Header().Set("Content-Type", "application/json")
        switch r.Method {
        case http.MethodGet:
            handleGetEmpresa(w, r, db)
        case http.MethodPost:
            handlePostEmpresa(w, r, db)
        case http.MethodPut:
            handlePutEmpresa(w, r, db)
        default:
            http.Error(w, "method not allowed", http.StatusMethodNotAllowed)
        }
    }
}

// GET por id via query param: r.URL.Query().Get("id")
// GET lista: retorna slice, nunca null — guard: if slice == nil { slice = []T{} }
// POST: json.NewDecoder(r.Body).Decode(&req) → validação → db.QueryRow INSERT RETURNING id → w.WriteHeader(201)
// PUT: req.ID obrigatório → db.Exec UPDATE SET ... updated_at = CURRENT_TIMESTAMP WHERE id = $N
// sql.NullString para colunas nullable; sql.NullFloat64 para NUMERIC nullable (preco dos planos)
```

`handlePutCliente`: Decode → req.ID obrigatório → UPDATE financeiro.clientes SET razao_social=$1, email=$2, telefone=$3, responsavel=$4, ativo=$5, updated_at=CURRENT_TIMESTAMP WHERE id=$6 → {"status": "updated"}.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP → handlers Go | Requests chegam sem garantia de payload válido; todo decode é seguido de validação |
| Handler → banco | Queries parameterizadas ($N); nunca interpolação de strings |
| Migration autorun | Migrations executadas no boot com privilégio de connection pool; IF NOT EXISTS protege idempotência |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | /api/financeiro/* endpoints | mitigate | withAuth(..., "admin") em todas as rotas; AuthMiddleware verifica JWT e role antes de chamar handler |
| T-02-02 | Tampering | handlePostCliente / handlePostContrato | mitigate | db.Begin() + defer tx.Rollback() garante atomicidade; falha parcial faz rollback completo |
| T-02-03 | Information Disclosure | handleListClientes com ILIKE | mitigate | ILIKE com parâmetro $N, nunca fmt.Sprintf com input direto; DISTINCT previne leakage por JOIN duplicado |
| T-02-04 | Denial of Service | ILIKE + GIN scan em tabela grande | accept | Índices GIN trigram minimizam custo; tabela tem dezenas de registros no v1, não centenas de milhares |
| T-02-05 | Elevation of Privilege | rotas financeiro.* | mitigate | withAuth exige role=="admin"; admin bypass é intencional (ver auth.go) — a role "admin" é concedida manualmente no banco |
| T-02-SC | Tampering | nenhum novo pacote npm/pip/cargo neste plano | accept | Plano usa apenas go.mod existente; sem novos pacotes — auditoria de pacotes não aplicável |
