<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → React SPA | Usuário interage com formulários — input não sanitizado antes de envio |
| React SPA → /api/financeiro/* | fetch() com JWT injetado pelo AuthContext monkey-patch |
| ProtectedRoute → /admin/login redirect | Usuário sem JWT válido é redirecionado antes de ver qualquer dado |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | /admin/financeiro/empresa (frontend route) | mitigate | ProtectedRoute verifica isAuthenticated via AuthContext; redirect automático para /admin/login se não autenticado |
| T-02-02 | Tampering | Formulários React (input fields) | mitigate | Campos required HTML5 + validação backend (handler Go retorna 400 para campos obrigatórios vazios); sem execução de input como código |
| T-02-03 | Information Disclosure | EmpresaPage — dados carregados no useEffect | accept | GET /api/financeiro/* requer JWT admin válido (backend); se token expirar, AuthContext redireciona para login; dados não são cacheados em localStorage |
| T-02-04 | Elevation of Privilege | ProtectedRoute | mitigate | ProtectedRoute é client-side guard — o backend tem a autoridade real (withAuth); mesmo que bypass ocorra no frontend, o backend retorna 401/403 |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | Nenhum novo pacote instalado neste plano — todos os imports de shadcn/ui e hooks já estão em package-lock.json |
</threat_model>
