<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /api/financeiro/* | Request HTTP cruzando do frontend React (ou curl) para o handler Go — input não confiável |
| JWT token → AuthMiddleware | Token recebido no Authorization header; claims não são confiáveis até validação |
| handler → PostgreSQL | Dados do request chegam ao banco via driver lib/pq |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | /api/financeiro/* endpoint | mitigate | withAuth(handler, "admin") obrigatório — AuthMiddleware valida JWT e role antes de executar handler; request sem token válido retorna 401 |
| T-01-02 | Tampering | SQL queries em financeiro.go | mitigate | Parameterized queries com $1, $2 em todos os db.QueryRow/db.Exec — sem string interpolation ou fmt.Sprintf com SQL |
| T-01-03 | Elevation of Privilege | /api/financeiro/* endpoint | mitigate | Role "admin" verificada em AuthMiddleware (auth.go:251-254); qualquer role diferente de admin recebe 403 |
| T-01-04 | Tampering | PUT /api/financeiro/empresa | mitigate | Handler verifica que req.ID não está vazio antes de executar UPDATE; sem ID → 400; UPDATE usa WHERE id=$N com parameterized value |
| T-01-05 | Information Disclosure | GET /api/financeiro/empresa | accept | Dados cadastrais (CNPJ, endereço) são informações da empresa proprietária do sistema — acessíveis apenas por admin autenticado; não contêm segredos de usuário |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | Esta fase não instala novos pacotes — todas as dependências já estão em go.mod e package-lock.json; nenhum checkpoint de slopcheck necessário |
</threat_model>
